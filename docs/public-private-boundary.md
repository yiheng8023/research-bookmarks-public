# Public / Private Boundary

## Public repository

Allowed:

- official source URLs;
- canonical documentation links;
- public taxonomy;
- import/export formats;
- source policy;
- validation scripts.

## Private overlay

Keep private:

- full browser exports;
- personal folders and labels;
- private notes and annotations;
- subjective scores or preference weights;
- non-official personal recommendations;
- account-specific or session-specific URLs.

## Promotion gate

Private-to-public promotion requires:

1. remove private notes and preferences;
2. confirm the source is official or clearly reviewable;
3. check license and provenance;
4. verify no account/session URL is present;
5. open a reviewed pull request.
