# research-bookmarks-public

[English](README.md) | 简体中文

这是一个公开安全的官方来源书签目录与分类种子，用于资源发现、知识图谱规划和摆脱浏览器绑定的书签可迁移管理。

## 仓库职责

本仓库是公开安全书签目录，提供可复用分类、官方来源目录和导入/导出约定，可作为更广义资源发现系统的输入。

它不是用户浏览器书签的完整导出。

## 本仓库提供什么

- 面向研究、工程、知识与工具资源的公开安全分类。
- 官方来源书签示例。
- 公开安全来源元数据验证。
- 私有书签 overlay 的边界模型。
- 给资源雷达使用的输入 lane。

## 本仓库不负责什么

- 个人浏览器书签。
- 浏览历史、私人笔记、私有文件夹、账号状态或个人偏好。
- 未经公开审查的非官方个人推荐。
- 资源评分、仓库生命周期分析或 GitHub 专项自动发现；这些由资源雷达负责。
- Agent Skill 准入审批；这由精选 Skills 仓负责。

## 与私有书签仓的关系

推荐模型：

```text
research-bookmarks-public
  -> 拥有公开安全分类、官方来源、导入/导出契约

private bookmarks repository
  -> 拥有完整个人书签、私人笔记、非官方资源、偏好和浏览器导出
  -> 可以消费公开分类
  -> 可以通过脱敏闸门把已审查官方来源提升回公开仓
```

不要盲目双向同步。私有到公开必须经过过滤和审查。

## 与资源雷达的关系

本仓库可以为资源雷达提供高层来源分类和官方入口。资源雷达仍负责发现、归一化、评分、生命周期状态、去重、报告和候选路由。

## 目录结构

```text
data/taxonomy.json                  公开分类种子
data/official-sources.example.json  官方来源示例
docs/                               边界、来源策略、同步模型
exports/                            导入/导出契约说明
scripts/verify.py                   结构与公开安全检查
```

## 验证方式

运行：

```bash
python -B scripts/verify.py
```

GitHub Actions 会在 pull request 和推送到 `main` 时运行同样的验证。

## 更新规则

1. 优先收录官方来源页和项目官方文档。
2. 不放入私有浏览历史、私人笔记、主观排名或个人文件夹。
3. 广义发现和生命周期自动化交给资源雷达。
4. 完整个人书签管理放在私有 overlay。

## 安全边界

本仓库按未来公开设计。任何会暴露个人偏好、私有上下文、非官方背书、账号状态或浏览行为的书签，都应留在私有 overlay。
